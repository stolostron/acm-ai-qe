How Z-Stream Analysis Works

Z-stream analysis classifies Jenkins pipeline test failures as PRODUCT_BUG, AUTOMATION_BUG, INFRASTRUCTURE, or NO_BUG through a 4-stage pipeline. Stage 1 gathers factual data from Jenkins, the cluster, and source repos. Stage 1.5 runs a comprehensive AI-driven cluster diagnostic. Stage 2 investigates each failure using a 12-layer diagnostic model with 5 MCP servers and classifies based on root cause. Stage 3 generates reports.

All data in this document comes from a real run against build #3913, tracing test RHACM4K-2576 from input to final classification.
