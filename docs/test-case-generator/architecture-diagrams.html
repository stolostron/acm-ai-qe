How the Test Case Generator Works

Generates Polarion-ready test cases from JIRA tickets through a 6-phase pipeline. Stage 1 gathers data deterministically via gh CLI. Phase 1 launches 3 parallel investigation agents (JIRA, code, UI). Phase 2 synthesizes findings with scope gating and AC cross-referencing. Phase 3 optionally validates on a live cluster. Phase 4 writes the test case. Phase 4.5 enforces a mandatory quality gate. Stage 3 generates Polarion HTML.

All data in this document reflects the actual pipeline implementation. Click any box in the diagrams below to see what each component does, its inputs/outputs, and the tools it uses.
