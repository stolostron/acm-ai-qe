Live Example: RHACM4K-2576 — Correctly Classified

All data shown in this document comes from a real analysis run against Jenkins build #3913 on cluster ashafi-aws-test. The traced test case is RHACM4K-2576 (legacy Carbon selector), which z-stream-analysis correctly classified as AUTOMATION_BUG with 90% confidence. Click any stage box to see what data was collected or produced at that point.

Key features by version: PR-5 data assertion pre-check, assertion analysis, failure mode categorization, backend API probes, graduated infrastructure scoring (v3.3). Backend probe source-of-truth validation with anomaly_source/classification_hint (v3.4). Environment Oracle (6-phase dependency health), Polarion test case context, KG-driven feature learning, PR-7 oracle dependency check (v3.5). Oracle pod-health fallback for CSV checks, managed cluster age filtering, DEFINITIVE signal requires 2+ confirmed failures, oracle signals are additive (not override) -- per-test evidence takes precedence (v3.5 fixes). External service failure detection, version compatibility constraints, known JIRA bugs cache (v3.5.1). Foundation and Install feature areas with Ginkgo test framework support, post-upgrade failure patterns, multi-pipeline repo mapping (v3.5.1 knowledge expansion). Comprehensive AI-driven cluster diagnostic (Stage 1.5) adapted from ACM Hub Health agent with 6-phase investigation, 10 diagnostic traps, baseline comparison, addon/webhook verification, self-healing knowledge, and classification guidance for improved INFRASTRUCTURE disambiguation (v3.6). 12-layer diagnostic investigation model (Compute→Control Plane→Network→Storage→Config→Auth→RBAC→API→Operator→Cross-Cluster→Data Flow→UI), per-group investigation agents for deeper root-cause analysis (v3.8). Provably linked grouping (Phase A4) using strict code-path criteria only, per-test verification within groups (4-point check), expanded counterfactual verification (D-V5) with 9 templates covering selector, button-disabled, timeout, data-assertion, blank-page, CSS, NetworkPolicy, operator, and ResourceQuota failures (v3.9). 60-file knowledge database with per-subsystem architecture, data-flow, and failure-signatures across 12 ACM subsystems + diagnostics methodology + healthy baseline + addon catalog + webhook registry + diagnostic traps. 5 MCP servers: ACM-UI (20), Jenkins (11), JIRA (25), Polarion (25), Neo4j (2). Verified ~91-100% accuracy across multiple builds.
