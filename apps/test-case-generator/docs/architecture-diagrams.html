Pipeline Overview

Generates Polarion-ready test cases from JIRA tickets through a 6-phase pipeline. Stage 1 gathers data deterministically via gh CLI. Phase 1 launches 3 parallel investigation agents (JIRA, code, UI). Phase 2 synthesizes findings with scope gating and AC cross-referencing. Phase 3 optionally validates on a live cluster. Phase 4 writes the test case. Phase 4.5 enforces a mandatory quality gate with MCP verification. Stage 3 generates Polarion HTML and validates conventions.

Invoke via /generate ACM-30459 or /generate ACM-30459 --version 2.17 --pr 5790 --area governance

Conflict Resolution Rules

UI elements (labels, routes, selectors): trust UI Discovery (reads source directly).
Business requirements (ACs, scope): trust Feature Investigator (reads JIRA).
What changed (files, diff): trust Code Change Analyzer (reads the diff).
Knowledge file authority: if ANY agent contradicts knowledge/architecture/<area>.md, trust the knowledge file until MCP verification resolves the conflict.
